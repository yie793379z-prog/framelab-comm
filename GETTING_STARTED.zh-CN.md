# FrameLab 新手从这里开始

这份指南写给**完全没有技术背景**、但想把 FrameLab 真正用起来的传播学、新闻学、媒体研究学生。

你不需要先懂：

- GitHub
- npm
- 前端开发
- API
- 数据库
- 项目源码

你只需要愿意照着步骤点一遍。

## 一、FrameLab 是什么？

你可以把 FrameLab 理解成：

**一个专门给文本做研究编码的小工作台。**

它帮你做的事情是：

1. 把文本放进来
2. 选择一个研究模板
3. 一条一条地编码
4. 看摘要
5. 导出成表格、报告或项目文件

它适合：

- 内容分析
- 框架分析
- 访谈前期整理
- 危机传播文本观察
- 课程作业
- 研究计划
- 论文前期材料整理

## 二、FrameLab 能做什么？不能做什么？

### 它能做什么

- 粘贴文本并拆成多个样本
- 导入 TXT / Markdown / CSV
- 对脏文本做保守清理
- 使用传播学模板做编码
- 人工修改所有字段
- 生成模拟或可选真实 AI 建议
- 批量为未编码样本生成建议
- 查看编码摘要
- 导出 CSV / Markdown / 编码手册 / JSON
- 用 JSON 重新打开项目
- 自动在本地保存当前工作区

### 它不能做什么

- 不能自动抓取微博、微信公众号或其他平台内容
- 不是爬虫
- 不会自动理解图片内容
- 不会替你做最终学术判断
- 不适合把 10000 条数据不经检查一键“自动分析完”

## 四、使用前你需要准备什么？

你需要：

- 一台电脑
- 浏览器
- 已安装 `Node.js`
- 一个终端窗口

### `Node.js` 是什么？

你可以把它理解成：

**让 FrameLab 能在你电脑上跑起来的基础环境。**

就像有些软件需要先装运行库，FrameLab 需要 Node.js 才能启动。

### 终端是什么？

终端就是一个可以输入命令的窗口。

- Mac：终端
- Windows：PowerShell / CMD

## 五、`npm install`、`npm run dev`、`localhost:3000` 是什么？

### `npm install`

这一步的意思是：

**“把 FrameLab 需要的小零件先装好。”**

通常第一次在这台电脑上运行时做一次就够了。

### `npm run dev`

这一步的意思是：

**“把 FrameLab 在我自己的电脑上临时打开。”**

它不会把项目发布到互联网，而是在你本地开一个网页服务。

### `localhost:3000`

你可以把它理解成：

- `localhost` = 这台电脑自己
- `3000` = 这个网页临时使用的门牌号

所以打开 [http://localhost:3000](http://localhost:3000)，就是在打开你自己电脑上的 FrameLab。

## 六、最简单的启动步骤

在项目文件夹里打开终端，然后输入：

```bash
npm install
npm run dev
```

然后在浏览器里打开：

[http://localhost:3000](http://localhost:3000)

## 七、默认语言和语言切换

FrameLab 默认界面是**简体中文**。

如果你想切换成英文：

1. 看页面顶部右侧的语言切换区
2. 点击 `EN`

如果想切回中文：

1. 点击 `简中`

系统会记住你的选择。

## 八、进入工作区后先做什么？

建议你第一次这样走：

1. 打开首页
2. 点击 `打开工作区`
3. 看见导入区以后，不要急着导数据
4. 先打开一个示例项目说明文件

最推荐先看：

- [examples/projects/wechat-news-cleaning-demo.md](./examples/projects/wechat-news-cleaning-demo.md)

## 九、如何粘贴文本？

如果你已经从网页、采访整理稿、公众号文章里复制了文本：

1. 把文本粘贴到导入框
2. 不同样本之间留一个空行
3. 看下方“检测到 X 条样本”
4. 如果数量合理，点击 `载入工作区`

### 一个空行是什么意思？

比如这样：

```text
第一段文本……

第二段文本……

第三段文本……
```

中间每一个空行，FrameLab 都会把它当成一条新样本。

## 十、如何清理复制出来很乱的文本？

如果你复制的是公众号文章、网页正文、宣传稿，经常会混进这些内容：

- `[图片]`
- `阅读原文`
- `点击关注`
- `广告`
- `免责声明`
- 多余空行

这时可以：

1. 把原文粘贴进导入框
2. 点击 `清理粘贴文本`
3. 自己再快速检查一遍
4. 没问题后再点 `载入工作区`

### 它会自动清理什么？

它只会保守处理明显噪音，例如：

- 重复空行
- 图片占位
- 明显独立存在的广告提示
- “阅读原文”“点击关注”这类独立提示

### 它不会做什么？

- 不会自动理解图片
- 不会自动判断什么句子最重要
- 不会替你决定文章真正的研究意义

所以：

**清理之后一定要自己再看一遍。**

## 十一、如何导入 TXT / Markdown / CSV？

### TXT / Markdown

适合：

- 访谈片段
- 手动整理的文本
- 多段声明
- 研究备忘

导入方法：

1. 点击 `选择文件`
2. 选择 `.txt` 或 `.md`
3. 看样本数量
4. 点击 `载入工作区`

TXT 和 Markdown 会按空行拆分样本。

### CSV

适合：

- 微博帖子
- 社交媒体数据表
- 新闻样本表格
- 你自己整理好的文本数据

FrameLab 会优先识别这些文本列名：

- `text`
- `content`
- `body`
- `message`
- `transcript`

如果你导入 CSV：

1. 点击 `选择文件`
2. 选择 `.csv`
3. 看系统识别到了哪个文本列
4. 看样本数是否合理
5. 点击 `载入工作区`

## 十二、怎么选对模板？

一个最容易记住的方法：

- 新闻报道 / 评论 /议题叙事：`新闻框架分析`
- 社交媒体帖子：`社交媒体内容分析`
- 访谈片段：`访谈预编码`
- 危机回应 / 公开声明：`危机传播扫描`

第一次用，不需要过度担心“是不是 100% 最优模板”。  
先选最接近的那个，把流程跑通，再决定要不要调整。

## 十三、完整推荐流程：从微信公众号示例走一遍

这是最推荐的新手完整试跑路径。

### 第 1 步：打开示例说明

打开：

- [examples/projects/wechat-news-cleaning-demo.md](./examples/projects/wechat-news-cleaning-demo.md)

### 第 2 步：复制那段“很乱”的公众号文本

在示例文档里找到：

- `一段“很乱”的原始复制文本`

把整段复制出来。

### 第 3 步：打开 FrameLab 工作区

1. 回到 FrameLab 首页
2. 点击 `打开工作区`

### 第 4 步：把乱文本粘贴进去

把刚才复制的文本粘贴到导入框。

### 第 5 步：点击 `清理粘贴文本`

系统会去掉明显噪音。

### 第 6 步：人工检查清理结果

重点确认：

- 标题还在
- 正文段落还在
- 与研究问题有关的句子没有被删掉

### 第 7 步：点击 `载入工作区`

现在样本会进入工作区。

### 第 8 步：选择 `新闻框架分析`

在模板区点击：

- `新闻框架分析`

### 第 9 步：先手动填，或者点“生成建议”

你可以二选一：

- 先自己填
- 或点击 `生成建议`

建议第一次这样做更稳：

1. 先手动选 `主导框架`
2. 再看 AI 给的建议是否有参考价值

### 第 10 步：人工复核字段

重点看：

- `主导框架`
- `问题界定`
- `建议的解决路径`

不要因为 AI 填了，就直接接受。

### 第 11 步：看 `编码摘要`

滚动到右侧的 `编码摘要` 面板。

你可以看：

- 总样本数
- 已编码样本数
- 完成度
- 各字段分布

### 第 12 步：导出 Markdown 报告

滚动到导出区，点击：

- `导出 Markdown`

这份文件适合：

- 课堂汇报
- 研究备忘
- 论文前期整理

### 第 13 步：导出编码手册

点击：

- `导出编码手册`

这份文件适合：

- 方法课作业
- 给老师展示你用了什么字段和选项

### 第 14 步：导出 JSON 项目

点击：

- `导出 JSON`

这一步非常重要，因为它可以让你之后继续打开同一个项目。

### 第 15 步：刷新页面

刷新浏览器。

此时你大概率会看到：

- `恢复上次工作区？`

### 第 16 步：先测试本地自动保存

点击：

- `恢复上次工作区`

如果内容恢复了，说明本地自动保存正常。

### 第 17 步：再测试 JSON 项目恢复

如果你还想测试“换一种方式恢复项目”：

1. 到导出区点击 `清除本地自动保存`
2. 刷新页面
3. 到“载入项目 JSON”
4. 点击 `选择 JSON 文件`
5. 选中刚才导出的 `framelab-project.json`
6. 确认替换当前工作区

如果项目回来，说明 JSON 恢复也正常。

## 十六、AI 建议应该怎么安全使用？

最重要的一句：

**AI 建议只是可编辑的辅助起点，不应被视为最终学术判断。**

### 默认情况

默认是模拟建议：

- 不需要 API key
- 更适合先熟悉流程

### 如果你启用了真实 AI

你要知道：

- 当前选中的样本文本会发送给你配置的提供方
- 可能产生费用
- 可能暂时失败
- 失败时 FrameLab 会自动回退到模拟建议

### 如果 Gemini 或真实 AI 失败怎么办？

你通常不需要慌：

1. 看界面提示
2. FrameLab 一般会自动改用模拟建议
3. 你的已编辑字段不会被覆盖
4. 可以稍后重试
5. 也可以暂时切回 mock 模式

## 十七、批量建议怎么用才稳妥？

FrameLab 支持：

- `为未编码样本生成建议`

但建议这样用：

1. 先手动做几条
2. 确认模板合适
3. 再对少量样本试批量建议
4. 跑完以后继续人工复核

请不要把批量建议理解成“自动完成研究”。

它更像是：

**先帮你铺一层草稿。**

## 十八、人工复核时重点看什么？

无论哪种模板，都建议重点检查：

- 字段是否真的贴合文本
- AI 是否过度自信
- 选项是否被误判
- 文本字段是否写得太空泛
- 关键引文是否真有研究意义

## 十九、编码摘要怎么看？

你可以把 `编码摘要` 当成：

**一个帮助你快速看整体分布的小看板。**

它适合看：

- 总样本数
- 已编码 / 未编码
- 完成度
- 单选、多选、布尔值、文本完成度、数值统计

它不适合替代真正的研究解释。

## 二十、导出文件怎么选？

### 导出 CSV

适合：

- Excel / WPS
- 继续表格整理

### 导出 Markdown

适合：

- 课堂展示
- 研究备忘
- 论文前期草稿

### 导出编码手册

适合：

- 方法课作业
- 说明你使用了什么字段和选项

### 导出 JSON

适合：

- 以后继续打开同一个项目
- 换一台电脑继续工作

## 二十一、本地自动保存怎么用？

如果你刷新页面，FrameLab 会尽量保留当前工作区快照。

你可能会看到：

- `恢复上次工作区？`

这时你可以：

- 点 `恢复上次工作区`
- 或点 `重新开始`

如果你想清掉浏览器里的本地快照，可以在导出区点：

- `清除本地自动保存`

## 二十二、绝对不要做的事

- 不要把 AI 建议当成最终学术结论
- 不要把敏感、隐私、可识别个人数据直接发给真实 AI 提供方
- 不要指望 FrameLab 自动抓取微博
- 不要指望 FrameLab 自动看懂图片
- 不要把没有清理过的海量社交媒体数据直接全量跑批量建议

## 二十三、如果你今天只想先完整跑通一次

请直接按这个顺序做：

1. 打开 [examples/projects/wechat-news-cleaning-demo.md](./examples/projects/wechat-news-cleaning-demo.md)
2. 复制“很乱”的公众号文本
3. 启动 FrameLab
4. 点击 `打开工作区`
5. 粘贴文本
6. 点击 `清理粘贴文本`
7. 人工检查
8. 点击 `载入工作区`
9. 选择 `新闻框架分析`
10. 手动填一部分，或点 `生成建议`
11. 人工复核
12. 查看 `编码摘要`
13. 导出 `Markdown`
14. 导出 `编码手册`
15. 导出 `JSON`
16. 刷新页面并测试自动保存
17. 再用 `JSON` 把项目重新载入

走完这 17 步，你就已经真正完成了 FrameLab 的完整主流程。
