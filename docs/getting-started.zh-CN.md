# FrameLab 零基础使用指南

这份指南写给**没有编程背景**、但想把 FrameLab 用起来的传播学、新闻学、媒体研究学生。

如果你只想知道“我到底该点哪里”，请直接看：

- [examples/README.zh-CN.md](../examples/README.zh-CN.md)
- [examples/projects/wechat-news-cleaning-demo.md](../examples/projects/wechat-news-cleaning-demo.md)
- [examples/projects/weibo-topic-analysis-demo.md](../examples/projects/weibo-topic-analysis-demo.md)
- [examples/projects/interview-coding-demo.md](../examples/projects/interview-coding-demo.md)

## 1. FrameLab 是什么？

你可以把 FrameLab 理解成一个**给文本做研究编码的小工作台**。

它不是聊天机器人，也不是写作润色软件。它更像是：

- 先把材料放进来
- 再按模板整理
- 然后逐条编码
- 最后导出成表格或报告

如果你做的是这些任务，FrameLab 会比较合适：

- 新闻框架分析
- 社交媒体内容分析
- 访谈前期整理
- 危机传播文本初步比对
- 课程作业
- 研究计划
- 论文前期资料整理

## 2. 它适合谁？

FrameLab 主要是给这些人准备的：

- 传播学学生
- 新闻学学生
- 媒体研究学生
- 刚开始做内容分析/框架分析的人
- 想先把研究流程跑通、而不是先学复杂软件的人

如果你不会写代码，也**完全可以用它**。你不需要理解项目源码，只需要会：

- 打开网页
- 选择文件
- 粘贴文本
- 点按钮

## 3. “本地运行”是什么意思？

FrameLab 目前主要是一个**本地优先**工具。

“本地运行”可以简单理解为：

- 这个工具先在你自己的电脑上启动
- 然后你在浏览器里打开它
- 数据默认保留在你自己的浏览器和本地文件中

它不是一个必须注册账号、登录云端数据库才能用的网站。

## 4. 第一次使用前，需要准备什么？

你需要：

- 一台电脑
- 已安装 `Node.js`
- 一个命令行窗口（Mac 上可以用“终端”，Windows 上可以用 PowerShell 或 CMD）

如果这些词你不熟悉，也不用紧张。你只需要知道两件事：

- `Node.js` 是让这个项目能跑起来的运行环境
- 终端就是一个输入命令的窗口

## 5. `npm install` 是什么意思？

第一次运行项目时，你会看到这条命令：

```bash
npm install
```

你可以把它理解成：

**“把这个项目需要的小零件先装好。”**

就像第一次做实验前，要先把剪刀、胶带、量尺、标签纸放到桌上。  
`npm install` 做的就是类似的事：把 FrameLab 运行所需的依赖装到本地。

通常只需要在第一次运行时做一次。

## 6. `npm run dev` 是什么意思？

第二条常见命令是：

```bash
npm run dev
```

你可以把它理解成：

**“把 FrameLab 在我自己的电脑上临时打开。”**

它不会把项目发到网上，而是在你电脑上启动一个本地网页服务，方便你在浏览器里查看和使用。

## 7. `localhost:3000` 是什么意思？

当你运行成功后，通常会看到：

```text
http://localhost:3000
```

这里的意思不是“别人都能看到这个网页”，而是：

**“这台电脑上的这个网页地址。”**

你可以把 `localhost` 理解成“我自己的电脑”，把 `3000` 理解成“这个网页临时开的门牌号”。

所以你只要在浏览器打开：

[http://localhost:3000](http://localhost:3000)

就能看到 FrameLab。

## 8. 最简单的启动步骤

在项目文件夹里打开终端，然后输入：

```bash
npm install
npm run dev
```

然后在浏览器访问：

[http://localhost:3000](http://localhost:3000)

## 9. 进入工作区之后先做什么？

建议第一次这样走：

1. 打开 `Workspace / 工作区`
2. 在导入区先选一个示例文件
3. 看系统识别出多少条样本
4. 点击“载入工作区”
5. 选择模板
6. 在左侧点开一个样本
7. 先手动填写几个字段
8. 再尝试 AI 建议
9. 最后导出 Markdown 或 CSV

## 10. 如何粘贴文本？

如果你已经从网页、文档、采访整理稿里复制了文本：

1. 把文本粘贴到导入框
2. 不同样本之间留一个空行
3. 看右下角的“检测到多少条样本”
4. 点击“载入工作区”

### 一个空行是什么意思？

比如你有三段新闻摘录：

```text
第一段新闻……

第二段新闻……

第三段新闻……
```

中间每一个空行，FrameLab 都会把它当成“新的一条样本”。

## 11. 如何导入文件？

FrameLab 现在支持这些文件：

- `.txt`
- `.md`
- `.csv`

### TXT / Markdown

适合：

- 访谈片段
- 手动整理的文本
- 研究备忘
- 多段声明

处理方式：

- 按空行拆成多个样本

### CSV

适合：

- 微博帖子
- 社交媒体数据表
- 新闻文本表格
- 你已经整理好的文本数据集

FrameLab 会优先识别这些文本列名：

- `text`
- `content`
- `body`
- `message`
- `transcript`

如果你有这些列名中的任意一个，通常就能直接导入。

## 12. 遇到复制出来很乱的文本怎么办？

比如微信公众号文章经常会带有这些东西：

- `[图片]`
- `阅读原文`
- `点击关注`
- 广告提示
- 免责声明
- 一堆多余空行

FrameLab 现在有一个保守的按钮：

- `清理粘贴文本`

它只会去掉**明显独立存在的噪音**。  
但它不是“自动理解文章内容”，所以你仍然要自己检查一遍。

特别注意：

- 图片里的意思不会自动进入文本分析
- 如果图片上的字对研究很重要，你需要手动把文字补进文本里

## 13. 怎么选模板？

最简单的记法：

- 新闻报道、评论、政策叙事：`News Framing Analysis`
- 微博/社交媒体帖子：`Social Media Content Analysis`
- 访谈摘录：`Interview Pre-coding`
- 危机回应、公开声明：`Crisis Communication Scan`

你不需要一次就选得很完美。  
先选一个最接近的模板，跑一遍，再根据研究问题微调。

## 14. AI 建议怎么用？

FrameLab 里的 AI 建议，最适合当作：

- 一个起步提示
- 一个“先帮我填个草稿”的辅助

不适合当作：

- 最终学术判断
- 不经检查直接引用的结论

### 最重要的一点

AI 建议只是可编辑的辅助起点，不应被视为最终学术判断。

### 默认情况

默认是**本地模拟建议**，不需要 API key。

### 如果你启用了真实 AI

只有当你自己配置了 OpenAI 或 Gemini API key 时，FrameLab 才会使用真实 AI。

这时你必须知道：

- 当前选中的样本文本会发送给对应 AI 提供方
- 可能产生费用
- 不要上传敏感、隐私、可识别个人信息

## 15. 手动修改编码是什么意思？

FrameLab 不是自动判定工具。

它的设计原则是：

- 你可以先手动填
- 也可以先看 AI 给的草稿
- 但最终每一项都应该由你自己检查和修改

如果你发现：

- AI 把框架选错了
- 访谈引文抓得太泛
- 危机回应策略判断太武断

就直接改掉，不需要“迁就”工具。

## 16. 怎么导出结果？

FrameLab 目前支持：

- `CSV`
- `JSON`
- `Markdown`
- `Codebook Markdown`

### CSV

适合：

- 放进 Excel / WPS 继续整理
- 做简单统计

### JSON

适合：

- 以后继续打开同一个项目
- 换台电脑继续工作

### Markdown

适合：

- 课堂汇报
- 研究备忘
- 论文前期草稿

### Codebook

适合：

- 方法课作业
- 给老师展示你使用的编码表

## 17. 怎么重新打开之前的项目？

如果你之前导出过 `framelab-project.json`：

1. 打开工作区
2. 滚动到导出区
3. 点击“选择 JSON 文件”
4. 选中之前导出的 `framelab-project.json`
5. 确认替换当前工作区

这样你之前的：

- 样本
- 编码结果
- 项目信息
- 自定义编码表

都会回来。

## 18. 什么情况下不建议直接上真实 AI？

这些情况请特别谨慎：

- 采访材料包含真实姓名
- 涉及未公开的私人聊天
- 包含电话号码、邮箱、住址等信息
- 涉及敏感群体或脆弱群体
- 你并不清楚 API 数据会怎么处理

更稳妥的做法是：

- 先用模拟建议或纯手动编码
- 先做匿名化
- 先做小样本试跑

## 19. 给第一次使用者的最简单建议

如果你今天只想用 15 分钟把这个工具走通，请这样做：

1. 导入 `examples/news_sample.csv`
2. 选择 `News Framing Analysis`
3. 手动编码 1 条
4. 对第 2 条试一次“生成建议”
5. 看“编码摘要”
6. 导出 `Markdown`
7. 导出 `Codebook`
8. 导出 `JSON`
9. 再把这个 `JSON` 载入回来

走完这一遍，你就已经掌握 FrameLab 的主流程了。
