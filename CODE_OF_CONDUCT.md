# Code of Conduct

## Our Commitment

FrameLab aims to be a welcoming project for students, researchers, educators, and open-source contributors.

We want participation in this project to be respectful, constructive, and beginner-friendly, regardless of:

- level of technical experience
- academic background
- language background
- nationality
- gender identity
- sexual orientation
- disability
- religion

## Expected Behavior

Please:

- be respectful and patient
- give useful, concrete feedback
- assume good intent where reasonable
- help newer contributors understand context
- discuss disagreements in a calm and evidence-based way

## Unacceptable Behavior

Please do not:

- harass or insult other contributors
- use discriminatory or hostile language
- attack people instead of discussing ideas
- deliberately derail discussions
- misrepresent fictional examples as real research data

## Research and Academic Discussion

This project touches communication research workflows. It is fine to disagree about methods, terminology, or template design, but those disagreements should stay specific, evidence-based, and respectful.

## Enforcement

Project maintainers may remove comments, close discussions, or reject contributions that do not follow this code of conduct.

## Scope

This code of conduct applies in:

- issues
- pull requests
- project discussions
- review comments

## Reporting

If you experience or observe unacceptable behavior, open a private channel with the maintainers when possible. If private reporting is not available yet, open a minimal issue asking for maintainer contact.

## Attribution

This document is a short project-specific adaptation of common open-source code of conduct practices, written to stay lightweight and readable for a student-facing project.
